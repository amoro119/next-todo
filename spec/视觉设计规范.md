# 视觉设计规范

## 1. 整体设计风格

- **设计理念**: brew-guide 极简 Apple 风格，干净、克制、注重内容
- **色彩倾向**: 中性色系，无暖色，以灰度为主构建界面层次
- **阴影**: 无偏移阴影，不模拟纸张堆叠效果
- **边框**: 统一 1px 实线边框 (`hsl(var(--border))`)，无 2px 粗边框
- **圆角**: 基础圆角 `0.5rem`（约 8px），对应 Tailwind 的 `rounded-lg`
- **背景纹理**: 可选微点阵背景（`.bg-pattern`），透明度 0.04，极淡装饰

整体追求「内容优先」的视觉秩序，所有 UI 元素服务于任务管理效率，不引入装饰性干扰。

## 2. 颜色体系

使用 CSS 自定义属性（HSL 格式）定义颜色，通过 `hsl(var(--name))` 引用。

### 浅色模式（`:root`）

| CSS 变量 | HSL 值 | 用途 |
|----------|--------|------|
| `--background` | `0 0% 98%` | 页面背景（近白色） |
| `--foreground` | `240 10% 3.9%` | 主要文字（深灰近黑） |
| `--primary` | `240 5.9% 10%` | 主要色 |
| `--primary-foreground` | `0 0% 98%` | 主要色上的文字 |
| `--secondary` | `240 4.8% 95.9%` | 次要背景 |
| `--secondary-foreground` | `240 5.9% 10%` | 次要背景上的文字 |
| `--muted` | `240 4.8% 95.9%` | 弱化背景（hover/禁用） |
| `--muted-foreground` | `240 3.8% 46.1%` | 弱化文字（占位符、次要信息） |
| `--accent` | `240 4.8% 95.9%` | 强调背景 |
| `--accent-foreground` | `240 5.9% 10%` | 强调文字 |
| `--destructive` | `0 84.2% 60.2%` | 删除/危险操作（红色） |
| `--destructive-foreground` | `0 0% 98%` | 危险按钮文字 |
| `--border` | `240 5.9% 90%` | 边框色（浅灰） |
| `--input` | `240 5.9% 90%` | 输入框边框 |
| `--ring` | `240 5.9% 10%` | 聚焦环（深灰） |
| `--radius` | `0.5rem` | 基础圆角 |

### 深色模式（`.dark`）

| CSS 变量 | HSL 值 | 用途 |
|----------|--------|------|
| `--background` | `240 10% 3.9%` | 页面背景（深黑） |
| `--foreground` | `0 0% 98%` | 主要文字（近白色） |
| `--primary` | `0 0% 98%` | 主要色 |
| `--primary-foreground` | `240 5.9% 10%` | 主要色上的文字 |
| `--muted` | `240 3.7% 15.9%` | 弱化背景 |
| `--muted-foreground` | `240 5% 64.9%` | 弱化文字 |
| `--accent` | `240 3.7% 15.9%` | 强调背景 |
| `--accent-foreground` | `0 0% 98%` | 强调文字 |
| `--destructive` | `0 62.8% 30.6%` | 危险操作（暗红） |
| `--border` | `240 3.7% 15.9%` | 边框色（深灰） |
| `--ring` | `240 4.9% 83.9%` | 聚焦环（浅灰） |

深色模式反转明度关系：背景变深色、文字变浅色，但色相保持在 240（蓝灰调），不引入暖色。

## 3. 字体规范

### 字体族

- **英文字体**: Inter（通过 `next/font/google` 加载，CSS 变量 `--font-inter`）
- **中文字体**: Noto Sans SC（通过 `next/font/google` 加载，CSS 变量 `--font-noto-sans-sc`）
- **后备字体**: `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`

### CSS 变量定义

```css
--font-sans: var(--font-noto-sans-sc), var(--font-inter), -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

### 字号层级

| 用途 | 大小 | Tailwind 类 |
|------|------|-------------|
| 基础正文 | 14px | `text-sm` |
| 小标题/强调 | 16px | `text-base font-semibold` |
| 主标题 | 36px | `text-4xl text-center` |
| 辅助文字 | 13px | `text-xs` |

Noto Sans SC 加载权重：400（常规）、500（中等）、700（粗体）。

### 应用方式

在 `layout.tsx` 中给 `<body>` 添加字体 CSS 变量类名，全局生效：

```html
<body class="[inter.variable] [notoSansSC.variable] font-sans antialiased">
```

## 4. 暗黑模式

### 切换机制

- 通过 `.dark` class 切换（挂载在 `<html>` 元素）
- CSS 中使用 Tailwind dark variant: `@custom-variant dark (&:where(.dark, .dark *))`
- 组件中条件式添加 `.dark` 类或使用 `dark:` 前缀

### ThemeProvider

- 位置：`app/providers/ThemeProvider.tsx`
- 初始化流程：
  1. 读取 `localStorage.getItem('theme')`
  2. 若有存储值，应用对应主题
  3. 若无，检查 `prefers-color-scheme: dark` 系统偏好
  4. 切换时同步写入 `localStorage` 和 `<html>` 的 class
- 提供 `useTheme()` hook：`{ theme: 'light' | 'dark', toggleTheme: () => void }`

### ThemeToggle

- 组件：`components/ThemeToggle.tsx`
- 位置：页面右下角固定（`fixed bottom-4 right-4 z-50`）
- 样式：圆形按钮（`p-2 rounded-full`），图标切换
  - 浅色模式 → 月亮图标（切换到深色）
  - 深色模式 → 太阳图标（切换到浅色）
- 按钮样式：`bg-background border border-border text-foreground hover:bg-muted transition-colors duration-150`

### 背景色

- 浅色模式背景：`hsl(0 0% 98%)` — 近白
- 深色模式背景：`hsl(240 10% 3.9%)` — 深黑
- 可选 `.bg-pattern` 微点阵纹理，作为固定定位背景层（`z-index: -1`，透明度 0.04）

## 5. 组件设计规范

所有组件使用 Tailwind utility class 构建，不再使用 `*.module.scss`。

### 普通按钮

```html
<button class="px-3 py-1.5 rounded-md border border-border text-sm hover:bg-muted transition-colors">
  按钮文字
</button>
```

### 主要按钮

```html
<button class="bg-foreground text-background hover:opacity-90 px-3 py-1.5 rounded-md text-sm transition-opacity">
  提交
</button>
```

### .btn 基础类

全局样式 `globals.css` 中定义了 `.btn` 和 `.btn-small` 类：

- `.btn`: `border-2` → `border border-border`, `border-radius: var(--radius)`, `padding: 8px 16px`, `transition: background-color 0.15s ease`
- `.btn-small:disabled`: `background-color: hsl(var(--muted))`, `cursor: not-allowed`, `opacity: 0.6`

### 输入框

```html
<input class="px-3 py-2 rounded-lg border border-border bg-background text-sm
             focus:outline-none focus:ring-1 focus:ring-ring" />
```

全局样式基线：`border-radius: var(--radius)`, `border: 1px solid hsl(var(--border))`, `padding: 8px 16px`。

### 模态框

```html
<div class="fixed inset-0 bg-black/40 z-50 flex items-center justify-center">
  <div class="bg-background border border-border rounded-lg p-6 min-w-[400px]">
    <!-- 内容 -->
  </div>
</div>
```

- 遮罩层：`bg-black/40`，覆盖全屏，居中内容
- 内容卡片：`rounded-lg` 圆角，`border-border` 边框，覆盖 `z-50`

### 卡片

```html
<div class="bg-background border border-border rounded-lg p-4">
  <!-- 卡片内容 -->
</div>
```

始终有 1px 边框，无阴影，无偏移效果。

### 其他全局样式

- `::placeholder`: 使用 `hsl(var(--muted-foreground))`，浅色模式为灰，深色模式自动适配
- `input:focus`, `select:focus`, `textarea:focus`: `outline: none`
- `button:focus`: `outline: none`
- `html`: `scrollbar-width: none`（隐藏滚动条），保留滚动功能

## 6. 动画规范

### 过渡时长

| 场景 | 时长 | 缓动 |
|------|------|------|
| hover 状态切换 | 150ms | `ease` |
| 按钮颜色过渡 | 150ms | `ease` |
| 列表项进出 | 250-400ms | `cubic-bezier(0.4, 0, 0.2, 1)` |
| 拖拽反馈 | 200ms | `ease` |

### CSSTransition 动画类

列表项进出场使用 `react-transition-group` CSSTransition，对应 `todo-fade`：

```css
/* 进入前 */  .todo-fade-enter       { opacity: 0; transform: translateY(20px); }
/* 进入中 */  .todo-fade-enter-active { opacity: 1; transform: translateY(0);
                 transition: 0.4s cubic-bezier(0.4, 0, 0.2, 1); }
/* 退出中 */  .todo-fade-exit-active  { opacity: 0; transform: translateY(-12px);
                 transition: 0.25s ease; }
```

### 拖拽反馈

```css
.drag-enter        { border-color: hsl(var(--ring)); }
.drag-enter-active { border-color: hsl(var(--ring));
                     background: hsl(var(--accent));
                     transition: 200ms ease; }
```

### 自定义关键帧动画

定义在 `globals.css` 中，通过类名或内联使用：

- `swing` — 摇摆效果（头部晃动）
- `popIn` — 弹出出现
- `goalItemFadeIn` — 目标项淡入
- `searchItemFadeIn` — 搜索结果淡入
- `highlightGlow` — 高亮发光脉冲
- `headShake` — 摇头抖动
- `modeTransition` — 模式切换过渡
- `confettiFall` — 彩带飘落（完成庆祝）
- `fadeIn` — 淡入
- `slideFromBottom` — 从底部滑入
- `slideIn` — 滑入
- `spin` — 旋转

## 7. 响应式设计

- 内容区最大宽度 700px（`.container`），在大屏上居中
- 宽屏模式（`.main-full-width`）最大宽度 1280px，用于目标等需要更多空间的视图
- 组件内部使用 Tailwind 响应式前缀，但不作为核心设计关注点——本应用主要为桌面使用优化
